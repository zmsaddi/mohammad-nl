## How to run migrations on production

After deploying v1.2.0 to Vercel:

1. Open https://console.neon.tech → SQL Editor
2. Connect to the **main** branch
3. Run the migration files in order:
   - `scripts/migrations/v1-2/001-profit-distribution-groups.sql`
   - `scripts/migrations/v1-2/002-entity-aliases-unique.sql`
4. After running both, create the `schema_migrations` table and record them:
   ```sql
   CREATE TABLE IF NOT EXISTS schema_migrations (
     id SERIAL PRIMARY KEY,
     version TEXT NOT NULL UNIQUE,
     filename TEXT NOT NULL,
     checksum TEXT NOT NULL,
     ran_at TIMESTAMPTZ DEFAULT NOW()
   );
   INSERT INTO schema_migrations (version, filename, checksum) VALUES
     ('001-profit-distribution-groups', '001-profit-distribution-groups.sql', 'f447270dbdfab064'),
     ('002-entity-aliases-unique', '002-entity-aliases-unique.sql', '<run checksum>');
   ```
5. Future migrations: `npm run migrate` (reads .env.test or .env.local)
